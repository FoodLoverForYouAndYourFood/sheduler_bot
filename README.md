TaskPair — минимальный Telegram-бот для трекинга задач. Хранит данные в SQLite, умеет приоритеты, дедлайны, DoD и утренний/вечерний отчёт.

## Запуск
- Python 3.11+.
- Установка: `python -m venv .venv && .venv\\Scripts\\activate && pip install -r requirements.txt`.
- Скопируйте `config.example.json` → `config.json`, заполните пользователей.
- Добавьте токен бота в `.env`: `BOT_TOKEN=...`.
- Запуск: `python bot.py`.

## Настройки `config.json`
- `report_time` / `evening_report_time` — время отчётов (`HH:MM`).
- `timezone` — таймзона, например `Europe/Moscow`.
- `users` — список пользователей. Поля:
  - `id` — Telegram ID,
  - `alias` — короткое обозначение,
  - `name` — отображаемое имя,
  - `is_admin` — `true`/`false`.

## Роли и доступ
- Не пользователь: не проходит проверку — бот пишет «нет доступа».
- Пользователь (`is_admin=false`): смотрит списки, добавляет задачи, меняет статус/дедлайн, получает отчёты.
- Админ (`is_admin=true`): всё выше + управление пользователями и удаление задач.

## Команды
Основные:
- `/add <текст>` — создать задачу (исполнитель → приоритет → дата → DoD).
- `/todo`, `/doing`, `/done` — списки по статусам.
- `/all` — все задачи.
- `/me` — задачи, назначенные на тебя.
- `/update <id> <todo|doing|done>` — сменить статус.
- `/deadline <id> <YYYY-MM-DD|clear>` — сменить/очистить дедлайн.
- `/report` — прислать отчёт сейчас.
- `/cancel` — отменить ввод задачи.

Админам:
- `/add_user <id> <alias> <Имя>` — добавить/обновить пользователя.
- `/add_admin <id> <alias> <Имя>` — добавить/обновить администратора.
- `/remove_user <id>` — убрать доступ (сбросит назначение в задачах).
- `/delete <id>` — удалить задачу.

## DoD (Definition of Done)
DoD — критерии, по которым задача считается готовой. Примеры:
- есть проверка/тест и понятно, как его запускать;
- задеплоено в нужное окружение;
- обновлены README/доки/скриншоты;
- уведомлены заинтересованные люди.

В диалоге `/add` бот спрашивает DoD после даты: введи текст или `skip`, чтобы оставить пустым. DoD показывается в карточке и в отчётах.
